Introduction — Why lists and tuples matter

Lists and tuples are two of Python's core ordered collection types. They let you group multiple values in a single variable and access them by position. This lesson focuses on practical usage: how to read and transform lists, when to pick tuples instead, and a compact methods reference you can use as a cheat sheet.

Day 7 — Harnessing Lists: Indexing, slicing, and core methods

Lists are mutable ordered sequences — you can add, remove, and reorder elements. Below you will find essential operations explained simply, followed by valid Python examples you can copy and run immediately.

Indexing and slicing — precise access

Indexing returns a single element by position; slicing extracts a sub-list. Indexes start at 0. Negative indexes count from the end. Step values let you sample items.

Python — indexing & slicing

# indexing and slicing examples
fruits = ['apple', 'banana', 'cherry', 'date', 'elderberry']
first = fruits[0]            # 'apple'
last = fruits[-1]            # 'elderberry'
middle_slice = fruits[1:4]   # ['banana', 'cherry', 'date']
start_to_third = fruits[:3]  # ['apple', 'banana', 'cherry']
step_slice = fruits[::2]     # every 2nd element: ['apple','cherry','elderberry']
print(first, last, middle_slice, start_to_third, step_slice)
  • fruits[0] — selects the first element (index starts at 0).
  • fruits[-1] — selects the last element; negative indices count from the end.
  • fruits[1:4] — slice from index 1 up to (but not including) index 4.
  • fruits[:3] — omitted start means start from index 0.
  • fruits[::2] — third parameter is step, here take every second item.

Common list operations (short examples)

These short, copy-pasteable snippets show common in-place operations and non-destructive alternatives where helpful.

Python — list ops

items = [3, 1, 4, 1, 5]

# append: add one item to the end
items.append(9)

# extend: add all elements from another iterable
items.extend([4, 2])

# insert: insert at index 0
items.insert(0, 7)

# remove: remove first occurrence of value
if 1 in items:
    items.remove(1)

# pop: remove and return last by default
last = items.pop()

# sorted: non-destructive sort (returns new list)
sorted_items = sorted(items)

# in-place sort & reverse
items.sort()
items.reverse()

print(items, sorted_items, last)
  • append(x) — add x to the end. Use when adding a single element.
  • extend(iterable) — append multiple elements from any iterable.
  • insert(i, x) — insert x before position i.
  • remove(x) — removes the first matching x; check membership to avoid ValueError.
  • pop([i]) — remove & return element at i; default removes last.
  • sorted() — returns a new sorted list; list.sort() sorts in-place.

Example: Grocery list — method-by-method sequence

This example walks through a grocery list step-by-step. Each numbered comment shows the method name and then the exact line of code that uses it. The code is valid Python — copy and paste into a REPL or script.

Python — grocery list demo (method sequence)

# 1. start with a basic list
grocery = ['milk', 'eggs', 'bread']

# 2. append() — add a single item to the end
grocery.append('butter')          # ['milk', 'eggs', 'bread', 'butter']

# 3. extend() — add multiple items from another list
grocery.extend(['apples', 'bananas'])  # adds apples and bananas

# 4. insert() — insert at a specific position (0 = front)
grocery.insert(1, 'orange juice')      # insert before index 1

# 5. remove() — remove by value (first occurrence)
grocery.remove('eggs')                 # removes 'eggs'

# 6. pop() — pop an item (default: last) and use it
item = grocery.pop()                   # removes and returns last item

# 7. index() & count() — find position and frequency
idx = grocery.index('bread')           # first position of 'bread'
cnt = grocery.count('milk')            # how many times 'milk' appears

# 8. sort() — keep list sorted for display (in-place)
grocery.sort()

# 9. reverse() — reverse ordering (in-place)
grocery.reverse()

# final view
print('final grocery:', grocery)
print('popped item:', item, 'bread index:', idx, 'milk count:', cnt)
  • Line-by-line: comments in the code show the purpose of each method; this sequence demonstrates typical list lifecycle: creation → add → modify → inspect → reorder.
  • Exceptions: remove(x) raises ValueError if x is absent. If unsure, check x in list first.
  • pop() is useful when you both want the value and to remove it from the list (e.g., treating list like a stack).

Day 8 — Lists & Tuples: Understanding structures and their functions

Tuples look like lists but cannot be changed after creation (they are immutable). That difference drives their use: tuples for fixed records, lists for mutable collections. Below are clear examples and patterns to help decide which to use.

Key differences (quick)

  • Mutability: lists are mutable; tuples are immutable.
  • Syntax: lists use []; tuples typically use () or commas (e.g., a = 1, 2).
  • Memory: tuples are slightly leaner in memory and can be marginally faster to create for fixed-size data.
  • Use cases: lists for dynamic, changeable collections; tuples for fixed records (e.g., coordinates, function argument packs, dictionary keys).

Tuples — examples and how to modify safely

Python — tuples demo

# tuple creation and immutability
coords = (35.6895, 139.6917)   # Tokyo lat/lon
pair = 1, 2                    # also a tuple

# tuples are immutable: the following would raise a TypeError
# coords[0] = 34.0

# to change values, convert to list, modify, then back to tuple
coords_list = list(coords)
coords_list[0] = 34.0
coords = tuple(coords_list)
print(coords)
  • Tuples are a good signal to readers: this data is not intended to change. Use them for fixed records.
  • Because they are hashable (when contents are hashable) you can use tuples as dictionary keys or set elements; lists cannot be used this way.

Tips & memory helpers — When to use lists vs tuples

  • Use lists when your collection will change (add/remove), or you need methods like append, pop, sort.
  • Use tuples when the data is fixed: they are good for lightweight records, function argument groups, and dictionary keys.
  • Memory & performance: tuples are slightly leaner; for very large fixed datasets tuples can reduce memory footprint.
  • Readability: choose the structure that communicates intent: tuple => fixed; list => dynamic.
  • Common pitfall: calling list methods on a tuple will raise AttributeError. Convert to a list if you need to change it.
